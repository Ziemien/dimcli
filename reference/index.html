Reference

Classes

Class Description
Cli Creates a handle to the shared command line configuration, this indirection allows options to be statically registered from multiple source files.
CliLocal Stand-alone cli instance independent of the shared configuration. Mainly for testing.
Cli::Opt<T> Metadata and reference to single value option.
Cli::OptVec<T> Metadata and reference to vector of values for multivalued options.

Configuration

Cli member functions

Function Description
opt<T> Add command line option
optVec<T> Add multivalued option
confirmOpt Add -y, --yes option that exits early when false and has an "are you sure?" style prompt when it's not present.
helpOpt Get reference to internal help option, can be used to change the description, option group, etc. To completely replace it, add another option that responds to --help.
passwordOpt Add --password option and prompts for a password if it's not given on the command line. If confirm is true and it's not on the command line you have to enter it twice.
versionOpt Add --version option that shows "${progName.filename()} version ${ver}" and exits. An empty progName defaults to argv[0].
group Changes config context to point at the selected option group of the current command, that you can then start stuffing things into.
title Heading title to display, defaults to group name. If empty there will be a single blank line separating this group from the previous one.
sortKey Option groups are sorted by key, defaults to group name.
command Changes config context to reference the options of the selected command. Use "" to specify the top level context. If a new command is selected it will be created in the command group of the current context.
action Action that should be taken when the currently selected command is run. Actions are executed when cli.exec() is called by the application.
header Arbitrary help text, for the command, before the usage section.
desc Help text, for the command, between the usage and arguments / options.
footer Help text, for the command, after the options.
helpCmd Add "help" command that shows the help text for other commands. Allows users to run "prog help command" instead of the more awkward "prog command --help".
helpNoArgs Adds before action that replaces the empty command line with "--help".
cmdGroup Changes the command group of the current command. Because new commands start out in the same group as the current command, it can be convenient to create all the commands of one group before moving to the next.
cmdTitle Heading title to display, defaults to group name. If empty there will be a single blank line separating this group from the previous one.
cmdSortKey Command groups are sorted by key, defaults to group name.
responseFiles Enabled by default, response file expansion replaces arguments of the form "@file" with the contents of the file.
envOpts Environment variable to get initial options from. Defaults to the empty string, but when set the content of the named variable is parsed into args which are then inserted into the argument list right after arg0.
imbue Change the locale used when parsing values via iostream. Defaults to the user's preferred locale (aka locale("")).
before Actions taken after environment variable and response file expansion but before any individual arguments are parsed.
iostreams Changes the streams used for prompting, printing help messages, etc. Mainly intended for testing. Setting to null restores the defaults which are cin and cout respectively.
conin Get current console input stream.
conout Get current console output stream.

Cli::Opt and Cli::OptVec member functions

Function Description
command Set subcommand for which this is an option.
group Set group under which this argument will show up in the help text.
show Controls whether or not the option appears in help pages.
desc Set description to associate with the argument in help text.
valueDesc Set name of meta-variable in help text. For example, in "--count NUM" this is used to change "NUM" to something else.
defaultDesc Set text to appear in the default clause of this options the help text. Can change the "0" in "(default: 0)" to something else, or use an empty string to suppress the entire clause.
defaultValue Allows the default to be changed after the opt has been created.
implicitValue The implicit value is used for arguments with optional values when the argument was specified in the command line without a value.
flagValue Turns the argument into a feature switch, there are normally multiple switches pointed at a single external value, one of which should be flagged as the default.
choice Adds a choice, when choices have been added only values that match one of the choices are allowed. Useful for things like enums where there is a controlled set of possible values.
clamp Forces the value to be within the range, if it's less than the low it's set to the low, if higher than high it's made merely high.
range Fail if the value given for this option is not in within the range (inclusive) of low to high.
require Causes a check whether the option value was set during parsing, and reports badUsage() if it wasn't.
prompt Enables prompting. When the option hasn't been provided on the command line the user will be prompted for it. Use Cli::fPrompt* flags to adjust behavior.
parse Change the action to take when parsing this argument.
check Action to take immediately after each value is parsed, unlike parsing itself where there can only be one action, any number of check actions can be added.
after Action to run after all arguments have been parsed, any number of after actions can be added and will, for each option, be called in the order they're added.

Parsing

Cli member functions

Function Description
parse Parse the command line, populate the options, and set the error and other miscellaneous state. Returns true if processing should continue.
resetValues Sets all options to their defaults, called internally when parsing starts.
fromString<T> Parses string into any supported type.
toString<T> Converts value of any supported type into a string.
badUsage Intended for use from return statements in action callbacks. Sets exit code (to EX_USAGE) and error msg, then returns false.
parseValue Used to populate an option with an arbitrary input string through the standard parsing logic. Since it causes the parse and check actions to be called care must be taken to avoid infinite recursion if used from those actions.
prompt Prompt sends a prompt message to cout and read a response from cin (unless cli.iostreams() changed the streams to use), the response is then passed to cli.parseValue() to set the value and run any actions.

Cli::Opt and Cli::OptVec member functions

Function Description
defaultFrom Default from to use when populating the option from an action that's not tied to a command line argument.
defaultPrompt Default name to use in prompts when not tied to a command line.
reset Set option to its default value.
parseValue Parse the string into the value, return false on error.
unspecifiedValue Set option (or add to option vector) to value for missing optionals.

After Parsing

Cli member functions

Function Description
exitCode EX_OK (0), EX_USAGE, or any value set by user defined actions.
errMsg Error message, only meaningful when exitCode() != EX_OK
errDetail Additional information that may help the user correct their mistake, may be empty.
progName Program name received in argv[0]
commandMatched Command to run, as selected by argv, empty string if there are no commands defined or none were selected.
exec Optionally parses, and then executes the action of the selected command; returns true if it worked. On failure it's expected to have set exitCode, errMsg, and optionally errDetail via fail().
fail Sets exitCode(), errMsg(), and errDetail(), intended to be called from command actions, parsing related failures should use badUsage().
commandExists Returns true if the named command has been defined, used by the help command implementation. Not reliable before cli.parse() has been called and had a chance to update the internal data structures.

Cli::Opt and Cli::OptVec member functions

Function Description
operator bool True if the value was populated from the command line, whether the resulting value is the same as the default is immaterial.
operator * Reference to underlying value or, for OptVec, vector of values.
operator -> Pointer to underlying value or value vector.
operator [] Array access to members of value vector (OptVec only).
from Name of the last argument to populated the value, or an empty string if it wasn't populated. For vectors, it's what populated the last value.
pos Absolute position in argv[] of last the argument that populated the value. For vectors, it refers to where the value on the back came from. If pos() is 0 the value wasn't populated from the command line or wasn't populated at all, check from() to tell the difference.
size Number of values, non-vectors are always 1.

Argv conversions

Cli static member functions

Function Description
